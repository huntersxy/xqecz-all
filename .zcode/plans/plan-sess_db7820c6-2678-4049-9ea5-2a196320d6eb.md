# Python 部署管理面板设计

## 架构

```
xqecz-panel/
├── server.py         # HTTP API 服务器
├── client.py         # CLI 客户端
├── deploy.py         # 部署逻辑
├── config.json       # 配置文件
└── requirements.txt  # 依赖
```

---

## HTTP API 服务器 (server.py)

使用 FastAPI 提供 REST API：

### 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/deploy | 一键部署 |
| POST | /api/build | 仅构建 |
| POST | /api/restart | 重启服务 |
| POST | /api/start | 启动服务 |
| POST | /api/stop | 停止服务 |
| GET  | /api/status | 服务状态 |
| GET  | /api/logs | 查看日志 |

### 认证

- API Key 认证（Header: `X-API-Key`）
- 配置文件存储 API Key

### 响应格式

```json
{
  "success": true,
  "message": "部署完成",
  "data": { ... }
}
```

## CLI 客户端 (client.py)

使用 click + httpx：

```bash
# 配置远程服务器
xqecz config set remote http://server:8080
xqecz config set api-key your-key

# 远程部署
xqecz deploy

# 远程重启
xqecz restart

# 查看状态
xqecz status
```

---

## 部署逻辑 (deploy.py)

```python
def deploy():
    1. git pull
    2. npm run build (frontend)
    3. go build (backend)
    4. 复制前端产物
    5. 发送 SIGINT 重启服务
```

## 使用方式

### 启动 API 服务器

```bash
cd xqecz-panel
python server.py
# 监听 0.0.0.0:9300
```

### CLI 调用

```bash
# 本地调用
python client.py deploy

# 远程调用
python client.py --remote http://server:9300 --api-key xxx deploy
```

### curl 调用

```bash
curl -X POST http://server:9300/api/deploy \
  -H "X-API-Key: your-key"
```
